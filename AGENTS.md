# AGENTS.md

## Scope

- 本文件用于约束本仓库内后续 Agent 的页面和交互改动。
- 在没有明确用户指示的情况下，默认优先保持现有功能、接口、页面映射和交互语义不变。

## Frontend Requirements

- 页面必须同时适配手机端和电脑端。
- 做页面样式或布局调整时，默认同时检查移动端和桌面端展示，不允许只针对单一端完成改动。
- 手机端保持相机应用式的底部导航体验，电脑端保持更宽的内容区域，不要把桌面端压成固定窄竖屏。
- 做 UI 重构时，优先调整布局和视觉层，不要随意改动业务逻辑、接口字段或已有控制流程。

## Navigation Mapping

- 底部标签当前固定为三个入口：
- `拍摄`：对应当前视频分析页面，组件为 `LivePusher`。
- `照片库`：当前未实现，点击后只提示“目前当前还未支持”，不要接入伪功能。
- `配置`：对应当前后台管理页面，组件为 `TemplateManager`。
- 标签切换必须使用“隐藏/显示”方式保留页面实例，不要通过卸载组件实现切换。

## Live Page Constraints

- `LivePusher` 页面承载腾讯直播 SDK、WebSocket 消息和视频分析交互，改动时默认视为高风险页面。
- 切换标签时不得卸载 `LivePusher`，否则会导致 WebSocket 连接和直播状态被中断。
- 在不影响现有功能的前提下再做样式优化，避免破坏：
- 推流、旁观、摄像头切换、通知弹窗、算法结果叠加、运动控制面板。
- “对准-人”页面目前只允许通过模版下拉选择参数，不再恢复为手填数值模式。
- 画面状态文案中，未开始推流或旁观时的“未连接”默认隐藏；做视觉调整时保留这一规则。
- 指令控制台默认折叠；若调整布局，保留“默认折叠、展开后功能不变”的行为。

## Template Management

- `TemplateManager` 页面职责是管理“对准-人”模版。
- 页面进入后默认查询模版列表，并支持新增、删除、修改、展开查看详情。
- 模版字段结构必须与“对准-人”当前使用的字段保持一致，不要单独发散出另一套字段定义。
- 当模版列表为空时，页面应稳定展示空状态，不应因为接口返回异常内容导致前端报错。
- 新增和编辑时，允许数字输入框出现临时空值，但提交时必须阻止空值提交。

## Service Endpoints

- 直播相关 WebSocket 地址默认保持为现有线上地址，除非用户明确要求修改。
- 运动控制下发地址默认保持为现有线上地址，除非用户明确要求修改。
- 配置服务地址默认保持为 `https://www.uiofield.top/config_server`，不要再回退到基于 `window.location` 的动态拼接。
- 如需改动任何线上地址、协议或路径，必须同步检查页面中所有使用点，避免出现部分页面仍指向旧地址。

## Safety Rules

- 不要为了“优化结构”而删除当前可用功能。
- 不要在未说明影响的情况下改动 WebSocket 消息格式、推流流程、模版字段名或页面入口映射。
- 如果改动可能影响手机端直播稳定性，优先选择保守方案，并在提交说明中明确写出影响范围。
